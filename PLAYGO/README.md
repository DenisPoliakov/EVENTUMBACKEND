# EVENTUM Mobile

`EVENTUM` состоит из трех рабочих частей:
- мобильное приложение на Flutter: `PLAYGO/frontend`
- админка: `PLAYGOADMIN/admin`
- единый backend и база: `PLAYGOADMIN/backend` + PostgreSQL

Мобильное приложение не поднимает свой отдельный сервер. Все данные оно получает из общего API на `http://localhost:4000`.

## Как это работает

Пользователь регистрируется и входит в приложение через backend `EVENTUM`.

После входа приложение работает с единым API:
- получает профиль пользователя
- показывает стадионы и матчи
- отправляет заявки на матч
- получает уведомления
- создает карточку футболиста
- работает с командами и приглашениями
- показывает вкладку `Игроки`

Если для пользователя включены ограничения, приложение получает их из того же backend и сразу меняет доступный функционал:
- блокировка платформы не дает войти
- блокировка заявок отключает подачу заявок на матч
- отсутствие карточки футболиста не позволяет заявляться на матч
- отсутствие команды также блокирует отправку заявки

## Логика карточки футболиста

Карточка футболиста создается в профиле пользователя.

Она хранит:
- позицию
- ведущую ногу
- рост, вес, возраст
- любимый формат игры
- короткое био
- аватар
- рейтинг
- до 3 сильных сторон
- до 3 статусов игрока

Правила:
- сильные стороны выбираются только из заранее заданного списка
- статусы игрока тоже выбираются только из заранее заданного списка
- если аватар не загружен, в списках и карточках можно использовать первую букву ника
- без карточки футболиста нельзя отправить заявку на матч

Во вкладке `Игроки` карточки доступны другим пользователям через поиск и фильтры.

## Логика команд

У пользователя в профиле есть блок `Ваша команда`.

Если команды нет:
- приложение показывает сообщение, что пользователь пока не состоит в команде
- пользователь может создать новую команду
- создатель команды автоматически становится капитаном

Если команда есть:
- капитан может приглашать игроков по логину или email
- приглашенный игрок получает уведомление и может принять или отклонить приглашение
- капитан может менять роль участника
- капитан может назначать игровую позицию в составе

Роли в команде:
- `CAPTAIN`
- `MEMBER`
- `SUBSTITUTE`

Игровые позиции состава:
- `GK`
- `DF`
- `MF`
- `FW`

Обычный участник команды не управляет составом и не видит управляющие действия капитана.

## Как работают заявки на матч

Заявка на матч создается не как произвольная форма, а от текущей команды пользователя.

Перед созданием заявки backend проверяет:
- у пользователя есть карточка футболиста
- пользователь не заблокирован на платформе
- у пользователя нет активного запрета на подачу заявок
- у пользователя есть команда
- матч существует и открыт для заявок
- у команды еще нет активной заявки на этот матч
- у капитана нет другой активной заявки на этот матч

После этого заявка попадает в backend и становится доступной в админке.

## Как работают автозаявки

У каждого матча есть режим модерации заявок:
- `MANUAL`
- `AUTO_FIRST_COME`

`MANUAL`:
- новая заявка получает статус `PENDING`
- дальше администратор вручную принимает или отклоняет ее

`AUTO_FIRST_COME`:
- backend считает количество уже одобренных команд
- если лимит `maxTeams` еще не заполнен, новая валидная заявка сразу получает `APPROVED`
- если лимит заполнен, новая заявка остается `PENDING`

Если одобренную команду сняли с матча:
- backend автоматически проверяет очередь `PENDING`
- первая подходящая заявка поднимается в `APPROVED`
